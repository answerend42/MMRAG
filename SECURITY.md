# 安全策略

MMRAG 当前是本地优先的研究和演示原型，不是面向公网的生产服务。请不要直接把当前版本暴露到公网环境。

## 支持范围

当前维护分支：

| 分支 | 状态 |
| --- | --- |
| `main` | 接收安全修复 |

## 报告安全问题

如果你发现安全问题，请不要在公开 issue 中披露完整利用细节。可以通过 GitHub 私有安全通道或仓库维护者可用的私下联系方式报告。

报告时请尽量包含：

- 受影响的文件或接口。
- 复现步骤。
- 潜在影响。
- 建议修复方向。

## 当前已知边界

- Web 服务默认只建议绑定 `127.0.0.1`。
- API 没有鉴权，默认用于本地演示。
- 上传图片会以 Data URL 进入请求体，当前只适合小规模本地测试。
- 图片文件接口会限制访问 PicRetrieve 图片根目录下的文件。
- 本地 LLM 和 PicRetrieve 数据由用户自行准备，仓库不托管模型权重或语料。

## 安全建议

- 不要把 `npm run app:dev` 启动的服务直接暴露到公网。
- 不要提交包含隐私或版权风险的图片语料。
- 不要把本地模型、索引、向量文件或 API key 放入 Git。
- 如果要部署到共享环境，请先补充鉴权、请求体限制、审计日志和文件访问策略。
